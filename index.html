1. Harvard PowerAgent and Power-Agent GitHub Ecosystem

Open-source agentic AI ecosystem for power systems
High source confidence Ecosystem Simulation tooling

PowerAgent is best understood as an open-source infrastructure ecosystem rather than a single end-user product. It organizes work around three layers: power/energy foundation models and datasets, MCP tool servers that let LLM agents operate power-system software, and workflow/skill layers that encode engineering procedures.

What It Mainly Does

  • Turns power-system software into tool-callable services for LLM agents.
  • Supports wrappers for PowerWorld, OpenDSS, PSSE, PSLF, PyPSA, pandapower, ANDES, Egret, PowerFactory, LTSpice, and related tools.
  • Provides workflow examples and skill playbooks for power-flow, contingency, dynamic, market, and mitigation studies.

How It Works

  • PowerMCP exposes native software operations through MCP servers.
  • LLM hosts call tool functions; solvers still do the physics and optimization work.
  • PowerSkills adds procedural guardrails: load/inspect/solve before advanced studies, then use mitigation playbooks for observed violations.
  • PowerWF adds LangGraph-style workflows, including a grid impact evaluation example for large load interconnection.

What Is Verifiable

  • Repository source and READMEs show active wrappers, example cases, and architecture patterns.
  • PowerFactory MCP exposes tools for load flow, short circuit, RMS simulation, parameter modification, CSV export, and study-case management.
  • PowerAgentBench defines a reproducible IEEE 39-bus steady-state task with constrained mitigations and scoring.

Limitations

  • Most value depends on local installation of licensed or specialist engineering software.
  • The ecosystem operates structured grid models; it does not validate applicant submission packages.
  • Public benchmarks are useful but narrow relative to production interconnection queues.
Difference from Grid Agents: PowerAgent gives agents hands and procedures for studies. Grid Agents needs to judge whether the submitted evidence package is complete, consistent, compliant, and ready for engineer review before or around studies.

Sources: PowerAgent site, Power-Agent GitHub, PowerMCP, PowerWF, PowerSkills, PowerAgentBench.

2. PowerWorkflow Grid Impact Evaluation

Agentic workflow for large-load grid impact studies
High source confidence Workflow Interconnection-adjacent

This is the closest source to interconnection work. It automates a technical grid impact study for a proposed large load such as a data center. The public workflow asks for a PowerWorld case file, a bus number, and load demand in MW, then runs baseline and modified studies.

User supplies PowerWorld file, bus, load MW
Baseline power flow
Baseline N-1 contingency
Add proposed load and rerun
Summarize violations and impact

What It Mainly Does

  • Automates a structured engineering study once the network model and interconnection location are known.
  • Uses PowerWorld tools for baseline power flow, contingency analysis, modified load analysis, and reporting.

How It Works

  • Uses LangGraph state transitions and GPT-4o tool calls.
  • Separates nodes for input collection, baseline analysis, contingency analysis, load modification, and interpretation.
  • Tool results are interpreted by the LLM, but the calculations are delegated to PowerWorld-facing functions.

Limitations

  • Assumes a usable PowerWorld model already exists.
  • Does not ingest multi-document applications or detect missing applicant evidence.
  • Input extraction in the public code is simple text parsing, not robust document understanding.

Source: PowerWF Grid Impact Evaluation.

3. PowerFM, OpenPowerBench, and Harvard Dataverse Dataset

Foundation-model catalog and benchmark data for power systems
High source confidence Benchmarks Datasets

PowerFM is a catalog-style repository for power and energy foundation-model work. OpenPowerBench is the concrete benchmark/data asset: a multi-task dataset for training and evaluating power-system foundation models across topology-dependent and topology-independent tasks.

What It Mainly Does

  • Provides benchmark data for load forecasting, price prediction, power flow, optimal power flow, contingency analysis, state estimation, and transient-state tasks.
  • Includes baseline code for LSTM, Transformer, and Time-LLM models on time-series tasks.
  • Links model/data work such as GridLDM, GridFM, mAIEnergy, and RAG-oriented tools.

How It Works

  • Topology-independent tasks use ISO time-series files such as CAISO, ERCOT, ISONE, MISO, NYISO, PJM, and SPP data.
  • Topology-dependent tasks use synthetic/standard network cases such as IEEE 39, IEEE 118, and IEEE 300 with steady and transient state files.
  • Evaluation logs include metrics such as normalized RMSE and R-squared for forecasting baselines.

What Is Verifiable

  • The Dataverse API returns title "OpenPowerBench", publication date May 12, 2025, version 4.1 released May 16, 2025, and CC0 1.0 licensing.
  • The GitHub repo includes dataset structure diagrams, training scripts, configuration files, and sample metrics logs.
Dataset note: the browser-rendered Dataverse page was not the best source in this environment, but the Dataverse API returned metadata and file listings directly.

Sources: PowerFM, OpenPowerBench GitHub, Harvard Dataverse DOI.

4. Andrea Pomarico Multi-Agent AI DSA Power Systems

Multi-agent dynamic security assessment in DIgSILENT PowerFactory
High source confidence PowerFactory Dynamic security

This repository is a Python multi-agent pipeline for RMS transient stability studies in DIgSILENT PowerFactory. It is not just a prompt wrapper: deterministic simulation and KPI extraction are implemented in code, while LLMs are used mainly for intake parsing and narrative/reporting layers.

Intake agent parses case
PowerFactory load flow and RMS simulation
CSV export and numerical KPIs
Plots, LLM summary, review
Word/PPT report and mitigation text

What It Mainly Does

  • Runs bus, line, and generator-switch fault scenarios in PowerFactory.
  • Computes voltage, speed, and rotor-angle KPIs from ComRes CSV exports.
  • Generates narrative reports, cross-case comparisons, mitigation recommendations, Word documents, and PowerPoint outputs.

How It Works

  • Uses the PowerFactory Python API and commands such as ComLdf, ComInc, ComSim, and ComRes.
  • LLM providers are Gemini and Groq; code separates fast and smart model aliases.
  • A mitigation agent receives structured grid data rather than trying to interpret a graph image.

Limitations

  • Runtime requires licensed PowerFactory, the correct local Python module path, and a matching project/study case.
  • Public source does not prove production utility adoption or acceptance of LLM-generated mitigation recommendations.
  • It automates studies, not submission-package completeness or applicant correspondence.

Sources: GitHub repo, Andrea Pomarico LinkedIn post.

6. enliteAI Energy and Maze RL

Reinforcement learning for topology control and grid optimization
Medium source confidence Operations RL

enliteAI is a Vienna-based AI company applying reinforcement learning to power-grid operations. Its public energy positioning focuses on congestion management, topology control, remedial-action support, outage planning, and operational grid security. This is closer to control-room and planning optimization than to interconnection application review.

What It Mainly Does

  • Trains RL agents against power-grid simulations to recommend operational actions.
  • Targets topology optimization, switching, bus splitting, congestion management, outage planning, and remedial-action curation.
  • Maintains Maze, an RL framework with PyTorch policies, graph/attention components, multi-agent support, and Gym-compatible environments.

How It Works

  • Uses simulation environments to evaluate candidate actions under security constraints.
  • Public papers include AlphaZero-style topology control and a two-phase multi-objective optimization method with TenneT data.
  • The 2025 TenneT/enliteAI paper reports day-ahead plans generated in 4-7 minutes, with high in-distribution success and lower out-of-distribution success.

Limitations

  • Public pages include marketing claims that are not all backed by accessible deployment evidence.
  • The TenneT work is strong technical evidence but still framed as research/planning support, not applicant package review.
  • Some experiments use simplified DC-flow approximations for tractability.

Sources: enliteAI, Energy solution page, Maze RL, 2022 AlphaZero topology paper, 2025 TenneT/enliteAI paper.
